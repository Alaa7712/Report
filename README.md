# نشر نموذج تقارير الأعطال على GitHub Pages

> هذا الدليل يوضح بالخطوات (مع صور توضيحية) كيفية رفع مشروعك وتشغيله كرابط عام.

## المتطلبات
- حساب GitHub مفعّل.
- مستودع جديد باسم `lab-report` (يمكن تغيير الاسم، فقط تذكري تعديله في الرابط).

## الملفات
- `index.html` — صفحة النموذج (RTL العربية).
- مجلد `images/` — صور توضيحية للخطوات.
- (اختياري) ملف `CNAME` إذا أردتِ نطاقًا مخصصًا.

## الخطوات

### 1) إنشاء مستودع جديد
- افتحي GitHub ثم اضغطي **New Repository** وأنشئي مستودعًا باسم `lab-report`.
![step1](images/step1.png)

### 2) رفع الملفات
- حمّلي الملف `index.html` والمجلد `images` إلى الفرع **main**.
![step2](images/step2.png)

### 3) تفعيل GitHub Pages
- اذهبي إلى **Settings → Pages**.
- اختاري **Source: main** ثم **Folder: /root**.
- اضغطي **Save**.
![step3](images/step3.png)

### 4) انتظار رسالة التفعيل
- ستظهر رسالة نجاح مع رابط موقعك.
![step4](images/step4.png)

### 5) فتح الرابط
- يكون الرابط عادة بالشكل:
```
https://username.github.io/lab-report/
```
![step5](images/step5.png)

### 6) تحديث المحتوى لاحقًا
- أي تعديل تدفعينه إلى الفرع **main** سيُحدث الموقع تلقائيًا.
![step6](images/step6.png)

---

> ملاحظة: إن أردتِ أن يكون الملف داخل مجلد `docs/` بدل الجذر، انقلي `index.html` إلى `docs/` ثم في صفحة **Pages** اختاري **Folder: /docs**.
